Welcome to check_graphite suite

Using graphite as the main data source for your monitoring leaves you with almost no tools to alert you on failure.

The combination of collectd agents running on your host and graphite/carbon/whipser on you monitoring server is small and also quite easy to setup.

If you're using icinga2 there is already some graphite support but only for storing performance data.

We would like to use graphite as a data source. Collectd e.g. pushes metric to graphite, like disk usage, swap, mem, load. On the monitoring server we can use a check such as check_graphite (several different options here) to query the data.

This is very unflexible. Imagine your data is spread over a few different counters and needs to be aggregated. E.g. load, with collectd there 3 values (1min,5min,15min as usual), check_graphite scripts usually can only test one value or sum or average multiple values.

The script in this repository are special scripts to query graphite. Because querying graphite is so easy, we just write a dedicated script for each check we like to have.

For example the check_df script pulls all disks and calculates the size.

These check are easy to integrate into icinga/nagios
